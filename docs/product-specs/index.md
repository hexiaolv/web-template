## 写作规范

每篇产品规格应包含：

1. **用户故事**：作为 [用户类型]，我希望 [功能]，以便 [价值]
2. **验收标准**：明确可测试的完成条件
3. **边界情况**：异常输入、权限边界、数据为空等
4. **不在范围内**：明确说明不做什么，避免误解

## 关键概念

（在此定义本产品的核心领域名词，避免歧义）

- **用户**：已登录的系统使用者
- **管理员**：`access === 'admin'` 的特权用户
