# 产品规格：多页签导航系统 (MultiTab)

> [技术设计](../design-docs/2026-04-23-multitab.design.md) | [实施路径](../implementations/2026-04-23-multitab.impl.md) | [避坑指南](../troubleshooting/2026-04-23-multitab.trouble.md)

## 1. 功能概述
为系统提供类似浏览器标签页的导航体验，支持页面状态保持、快速切换及灵活的页签管理。

## 2. 功能清单
- **页签管理**：
  - 点击左侧菜单自动开启或激活对应页签。
  - 限制最大开启数量为 15 个，超出时弹出提示。
  - 支持固定页签（如“工作台”），固定页签不可关闭。
- **状态保持 (Keep-Alive)**：
  - 在页签间切换时，保留页面的滚动位置、表单输入、查询条件等状态。
  - 仅在点击“刷新”或关闭后再重新打开时重置状态。
- **多维度关闭**：
  - 支持单个关闭（× 按钮）。
  - 支持右键菜单：关闭其他、关闭右侧、关闭全部、刷新当前页。
- **UI 交互**：
  - 页签溢出时支持左右滚动。
  - 激活页签时，自动将其滚入可视区域。
  - 适配 Side、Top、Mix 三种布局模式。

## 3. UI 规范
- **高度**：固定为 40px。
- **样式**：采用 Ant Design 风格，支持暗色模式自适应。
- **定位**：在 content 内容区域顶部粘性定位 (Sticky)。

## 4. 全局配置
可以通过 `config/defaultSettings.ts` 中的 `tabsLayout` 字段控制开关：
- `tabsLayout`: 是否开启多页签系统（`true`/`false`）。
- `homeTabPath`: 设置首页页签的路由路径（例如 `/` 或 `/dashboard/workplace`）。系统会自动探测其背后的物理组件，确保 `/` 及其重定向目标不产生重复页签。

## 5. 核心算法：物理组件去重
系统采用“物理组件指纹”去重算法，而非简单的路径字符串匹配：
- **原理**：基于 Umi 的路由表探测每个路径对应的 `.tsx` 组件文件。
- **优势**：完美解决 `/` 重定向到业务页面（如 `/welcome`）时产生的页签重复问题；同时支持多个路由别名指向同一个物理页面的自动合并。

## 6. 交互特性
- **拖拽排序 (Dnd-Kit)**：
  - **水平限制**：页签仅允许在水平轴向移动。
  - **垂直出界保护**：当鼠标垂直方向移出页签栏区域（阈值 60px）时，系统会自动停止碰撞检测，防止页签误位移。
- **固定页签 (Pin/Unpin)**：
  - **首页强制固定**：首页默认固定在最左侧，禁止拖拽和关闭，且不显示固定图标。
  - **手动固定**：用户可通过右键菜单“固定页签”。固定后的页签会自动移动到左侧固定区，禁止关闭，并显示“大头针”标识。
- **右键上下文菜单**：
  - 提供**刷新、固定/取消固定、关闭其他、关闭右侧、关闭全部**等便捷操作。
  - 系统会根据当前页签状态（是否为首页、是否已固定）动态调整菜单项的可用性。
