# 技术设计：多页签导航系统 (MultiTab)

## 1. 架构选型
本系统采用 **DOM 级缓存 (Display Control)** 方案，而非 React 状态提升方案。

### 为什么选择 DOM 级缓存？
- **零外部依赖**：无需引入 `react-activation` 等复杂外部库。
- **状态兼容性**：完美保留 `useModel`、`useRef`、滚动条位置及第三方组件的内部 DOM 状态。
- **性能平衡**：对于中后台 15 个以内的页签，内存开销处于可控范围内。

## 2. 核心原理
### 2.1 状态驱动 (multiTab Model)
使用 `@umijs/max` 的 `useModel` 管理 `tabs` 数组。

### 2.2 视图持有 (KeepAliveWrapper)
通过 `display: block/none` 控制页面容器的可见性：
- 当切换路由时，新的 `children` 被捕获并存入 `cacheRef`。
- 渲染器遍历 `tabs`，为每个路径创建一个持久化的 `div` 容器。

### 2.3 路由同步 (RouteListener)
监听 `useLocation` 变化，自动触发 `openTab`。通过 `useIntl` 动态解析 `menu.xxx` 配置，确保页签标题与菜单完全一致。

## 3. 布局兼容逻辑
页签栏需要根据 `layout` 配置动态调整 `z-index` 和 `top` 偏移量：
- **Side/Mix 模式**：`z-index` 设为 9，避免遮挡 Header 的层级（100+）。
- **FixedHeader 开启**：页签栏 `top` 自动偏移 `56px`。
