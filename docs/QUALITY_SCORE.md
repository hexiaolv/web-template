# QUALITY_SCORE.md

> 质量标准定义。代码合并前对照此文档自查。

## 评分维度

### 1. 类型安全（30分）

| 检查项 | 分值 |
|--------|------|
| 0 处 `any` 使用 | 15 |
| 所有 Props 有 interface 定义 | 8 |
| 类型导入用 `import type` | 4 |
| API 响应有泛型类型 | 3 |

### 2. 代码规范（25分）

| 检查项 | 分值 |
|--------|------|
| Biome lint 0 错误 | 15 |
| 命名符合规范（PascalCase/kebab-case/camelCase） | 5 |
| 导入顺序正确 | 3 |
| 无无效注释 | 2 |

### 3. 可靠性（25分）

| 检查项 | 分值 |
|--------|------|
| 加载/错误/空状态都有处理 | 12 |
| 破坏性操作有二次确认 | 8 |
| API 调用有 try/catch 或 onError | 5 |

### 4. 可维护性（20分）

| 检查项 | 分值 |
|--------|------|
| 组件职责单一（< 200 行） | 8 |
| 无重复代码（DRY） | 6 |
| 测试覆盖关键逻辑 | 6 |

## 合并门槛

- **生产功能**：总分 ≥ 85
- **修复/小改动**：总分 ≥ 70
- **原型 / 实验性**：无强制要求，但需注明

## 合并前自查命令

```bash
pnpm biome        # 格式化
pnpm biome:lint   # 规范检查
pnpm tsc          # 类型检查（0 错误）
pnpm test         # 测试通过
```
