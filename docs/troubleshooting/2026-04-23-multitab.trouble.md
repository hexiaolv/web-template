# 避坑指南：多页签导航系统 (MultiTab)

## 1. 登录后白屏 (Redirect Interception)
**方案**：在 `KeepAliveWrapper` 中增加“旁路渲染”逻辑：如果当前路径不在页签列表中，直接渲染 `children`。

## 2. 刷新后标题变为原始 Key / 控制台警告 (i18n Sync)
### 症状
- 在“二级管理页”刷新浏览器，页签标题变为 `sub-page`。
- 控制台报错：`[React Intl] Missing message: "menu.sub-page"`。
### 原因
`RouteListener` 在尝试多级翻译时，如果请求了不存在的 Key（如 `menu.sub-page`），`intl` 会打印警告。
### 解决
在 `RouteListener` 中集成 `useIntl`，并在调用 `formatMessage` 前通过 `key in intl.messages` 校验 Key 的存在性，确保只有存在的 Key 才会触发翻译，否则静默回退到原始名称。

## 3. UI 元素遮挡 (z-index Conflict)
**方案**：将页签栏 `z-index` 调整为 `9`，确保其在 Header 和侧边栏操作按钮下方。

## 4. 样式冲突 (Global CSS Override)
**方案**：删除 `TabBar.less` 中的硬编码背景色覆盖，由全局主题变量控制。
